# Claude Code 插件全面指南：命令、子代理和 Hooks

## 什么是 Claude Code 插件？

Claude Code 插件是一种自定义功能集合，包含以下四个核心组件：
- **Slash 命令 (Slash Commands)**：自定义的斜杠命令，用于快速执行特定任务
- **子代理 (Subagents)**：专门的 AI 代理，用于处理特定领域的任务
- **Hooks**：在开发生命周期特定点自动执行的 shell 命令
- **MCP 服务器**：扩展 Claude Code 功能的外部服务

## Claude Code 插件系统概述

### 插件安装方式
- 通过 `/plugin` 命令安装
- 可以开启/关闭以减少系统提示复杂性
- 在终端和 VS Code 中均可使用
- 支持团队级分发和管理

### 插件配置
- 基本插件在 `.claude-plugin` 目录中配置清单文件
- 通过本地市场进行测试
- 支持全局和项目级配置

## 命令 (Commands) 详细分析

### 1. `/vita-core:code_review` - 多方面代码审查

**功能**：使用并行的 code-review-expert 代理进行多方面代码审查

**参数**：`[what to review]` - 例如 "recent changes", "src/components", "*.ts files", "PR #123"

**核心特性**：
- **5个专业审查方面**：
  - 架构与设计审查
  - 代码质量审查
  - 安全与依赖审查
  - 性能与可扩展性审查
  - 文档与 API 设计审查

- **智能策略**：
  - 根据文件类型自动选择相关审查代理
  - 支持并行执行以提高效率
  - 提供端到端影响分析

- **输出格式**：
  - 关键问题分级（🔴严重、🟠高优先级、🟡中等优先级、🟢低优先级）
  - 具体文件位置和代码示例
  - 质量指标评分表
  - 系统性问题识别

### 2. `/vita-core:explore` - 深度代码库探索

**功能**：使用多个并行探索代理进行深度代码库探索

**参数**：`<what to explore in the codebase>`

**查询分类策略**：
- **广度优先探索**：覆盖多个独立领域，系统概览
- **深度优先探索**：单个功能的深入理解，实现细节
- **简单查找**：特定类/函数/变量的精确定位

**执行流程**：
1. **查询分类**：确定探索策略和资源分配
2. **并行执行**：同时启动多个探索代理
3. **文件系统聚合**：从 `/tmp/explore_*.md` 文件收集结果
4. **综合报告**：生成统一的探索报告

**输出示例**：
- 执行摘要
- 关键位置（文件:行号）
- 详细上下文和代码片段
- 文件索引和调用图

### 3. `/vita-core:research` - 深度研究

**功能**：使用多个并行子代理进行深度研究

**参数**：`<question to investigate>`

**查询类型**：
- **广度查询**：多个独立方面的调查
- **深度查询**：单个主题的深入理解
- **简单事实查询**：快速事实验证

**研究流程**：
1. **查询分类和策略制定**
2. **并行研究执行**（2-10个子代理）
3. **文件系统聚合**（`/tmp/research_*.md`）
4. **最终报告合成**

**质量保证**：
- 权威来源优先级排序
- 交叉引用验证
- 矛盾信息记录
- 研究方法论说明

### 4. `/vita-core:pr-summary-cn` - 中文 PR 摘要

**功能**：从 git 变更生成简洁的中文 PR 摘要

**参数**：`[base-branch]`（默认为 master）

**生成内容**：
- 变更摘要（一句话概括）
- 主要变更（3-5个重要变更点）
- 技术细节（必要的架构或实现说明）

**数据来源**：
- 当前分支状态
- 从基础分支开始的提交历史
- 详细的代码变更统计
- 具体的代码差异

## 子代理 (Subagents) 详细分析

### 1. `code-review-expert` - 代码审查专家

**专业领域**：全面的代码审查专家，覆盖5个核心方面

**核心能力**：
- **上下文感知审查**：理解项目约定和架构模式
- **模式识别**：检测项目特定模式和反模式
- **深度根因分析**：提供表面问题→根因→解决方案的完整分析
- **跨文件智能**：分析文件间关系和依赖
- **进化式审查**：跟踪代码随时间的演变

**审查重点**：
1. **架构与设计**：模块组织、关注点分离、设计模式
2. **代码质量**：可读性、命名、复杂性、DRY 原则
3. **安全与依赖**：漏洞、认证、依赖管理、供应链
4. **性能与可扩展性**：算法复杂性、缓存、异步模式
5. **文档与 API**：README、API 文档、破坏性变更

**输出模板**：
- 执行摘要
- 分级问题列表
- 质量指标评分
- 优势和建议
- 系统性模式识别

### 2. `explore-agent` - 探索代理

**设计目标**：专业的代码库上下文检索代理

**核心特性**：
- **并行探索规划**：每轮最多8个并行bash工具调用
- **高效搜索策略**：使用 grep、find、cat 等高效命令
- **结果排序和过滤**：按相关性和文件位置排序
- **结构化报告**：机器和人类代理均可理解的格式

**工作流程**：
1. **并行探索规划**（最多4轮）
2. **并行工具调用执行**
3. **结果排序和过滤**
4. **上下文报告构建**
5. **迭代探索**（如需要）

**输出格式**：
```
[CONTEXT REPORT]
Query: "查询内容"
Files matched:
  - <filename>:<start_line>-<end_line>
Code snippets:
  - "代码片段"
High-level summary:
  - 单行摘要
```

### 3. `research-agent` - 研究专家

**专业领域**：专门的信息收集研究专家

**任务模式识别**：
- **快速验证模式**（3-5个工具调用）：事实核查
- **专注调查模式**（5-10个工具调用）：特定方面深入研究
- **深度研究模式**（10-15个工具调用）：完整主题理解

**搜索策略**：
1. **初始广泛搜索**：了解领域概况
2. **针对性深度挖掘**：跟踪有希望的线索
3. **空白填补**：解决特定未覆盖的方面

**来源质量层次**：
1. **主要来源**：原始研究、官方文档
2. **学术来源**：同行评议论文
3. **专业来源**：行业报告、技术文档
4. **新闻来源**：权威新闻机构
5. **一般网络**：博客、论坛（谨慎使用）

**输出策略**：
- 完整报告保存到 `/tmp/research_[timestamp]_[topic].md`
- 返回轻量级摘要
- 包含来源引用和研究方法论

## Hooks（当前配置中未发现）

Hooks 是在开发生命周期特定点自动执行的 shell 命令。虽然当前 Vita 插件中没有配置 Hooks，但典型的 Hook 类型包括：

### 8种 Hook 类型
1. **UserPromptSubmit**：用户提交提示时
2. **ToolCallStart**：工具调用开始时
3. **ToolCallEnd**：工具调用结束时
4. **AgentStart**：代理启动时
5. **AgentEnd**：代理结束时
6. **CommandStart**：命令开始时
7. **CommandEnd**：命令结束时
8. **SessionStart**：会话开始时

## 使用建议

### 最佳实践
1. **命令选择**：
   - 代码审查使用 `/vita-core:code_review`
   - 代码库探索使用 `/vita-core:explore`
   - 外部研究使用 `/vita-core:research`
   - PR 摘要使用 `/vita-core:pr-summary-cn`

2. **并行处理**：
   - 所有命令都支持并行代理执行
   - 大幅提高处理效率
   - 避免顺序执行的延迟

3. **文件系统利用**：
   - 探索和研究命令将详细结果保存到 `/tmp/` 目录
   - 主报告返回简洁摘要
   - 需要时可以深入查看完整报告

### 集成策略
- **探索命令**用于内部代码库理解
- **研究命令**用于外部文档和API参考
- **结合使用**在实现需要外部库知识和内部代码变更的功能时

## 总结

Vita 插件提供了全面的开发工具集：

1. **代码审查**：多角度、深度的代码质量分析
2. **代码库探索**：高效的上下文检索和代码理解
3. **外部研究**：权威信息的收集和分析
4. **PR 摘要**：中文环境下的变更总结

这些工具协同工作，为开发者提供了从代码理解到质量保证，从内部探索到外部研究的完整工作流支持。通过并行处理和智能代理设计，显著提高了开发效率和代码质量。
